how rarity is calculated & how to verify it

rarity scores use the industry-standard statistical rarity method: each trait's score is 100 / trait_supply_percentage. rarer traits score higher. a token's total score is the sum of all its trait scores. the same formula is used by rarity.tools and rarity sniper.

trait score  =  100 ÷ % of collection that has this trait
token score  =  sum of all 8 trait scores
rank         =  position when all 4,800 tokens sorted by score (desc)
01
read the contract
the goblynz smart contract is publicly deployed on ethereum mainnet. all token metadata is stored permanently on arweave — not a server we control.
view contract on etherscan ↗
02
fetch any token's metadata
after reveal, call tokenURI(id) on the contract to get the arweave URL. open it — you'll see the raw JSON with every trait and its value.
arweave gateway ↗
03
compute the score yourself
take each trait value, look up what % of the collection has it, apply 100/pct, add all 8 up. you'll get exactly what this app shows. no black box.
04
cross-check with rarity tools
once the collection is indexed, rarity.tools and rarity sniper will show the same scores. if they differ, the formula and source data will be published openly for audit.
